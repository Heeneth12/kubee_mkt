3 Approaches for the Kubee Marketing Site

Each handles routing, structure, and conversion differently. Pick the one that fits best.

A

App Router + Static Pages (Recommended)

Next.js 16 App Router with each page as a static route: /, /pricing, /blog, /contact, /about, /terms, /privacy. Blog uses /blog/[slug] dynamic route with MDX or JSON content. All pages statically generated (SSG) for best SEO. Shared layout with navbar + footer. Tailwind CSS v4 throughout.

Pros

  • Best SEO — fully static HTML
  • Fast page loads
  • Clean URL structure
  • Easy to extend

Cons

  • Blog needs rebuild on new posts (unless using ISR)
B

Single-Page with Scroll Sections

One long homepage with all sections (hero, features, pricing, testimonials, contact). Separate pages only for legal (terms, privacy). Blog is an external link (e.g., Medium or Notion). Simpler to build and maintain.

Pros

  • Faster to build
  • Smooth scroll UX
  • Less maintenance

Cons

  • Weaker SEO (fewer pages to index)
  • Can't blog natively
  • Hard to scale
C

App Router + CMS-backed Blog (Contentlayer/MDX)

Same as Option A but the blog is powered by local MDX files with Contentlayer for type-safe content. Blog posts live in /content/blog/*.mdx. Ideal if you plan to publish content regularly for SEO. More setup upfront but very powerful long-term.

Pros

  • Best long-term SEO content engine
  • Type-safe, versioned blog content
  • No external CMS needed

Cons

  • More setup
  • Non-technical editors need Git

⭐ My recommendation: Option A — it's the right balance for an SEO-first marketing site. Blog can start simple (JSON/hardcoded posts) and evolve to MDX later. Fast to build, great Google performance.
